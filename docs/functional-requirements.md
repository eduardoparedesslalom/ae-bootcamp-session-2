# Functional Requirements

This document lists the core functional requirements for the To Do App.

## FR-1: Edit a Task

- The user must be able to edit the name/text of an existing task.
- The edit action should be accessible directly from the task list item.
- Changes must be persisted to the backend database.
- The user should be able to cancel an edit without saving changes.

## FR-2: Sort Tasks via Drag Handle

- The user must be able to reorder tasks in the task list using a drag-and-drop handle.
- The drag handle must be located on the right side of the DELETE button for each task item.
- The new order must be reflected immediately in the UI.
